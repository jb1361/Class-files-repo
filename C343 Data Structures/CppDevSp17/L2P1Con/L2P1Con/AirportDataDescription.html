Airport Data

Last Updated: 10/1/2016

Airport Data Overview:

  • The Airport Data files contain data for airports related to the name, code, location, and altitude

Airport Record Format in Data Set:

There are 13 fields in each record, in the order listed below, and the field's data type is included:
  1. code - (Text) The airport's International Air Transport Association's 3-letter code
  2. name - (Text) Airport's name
  3. city - (Text) City where airport is located
  4. country - (Text) Country where airport is located
  5. latitude degree - (Integer) Airport's degrees latitude
  6. latitude minutes - (Integer) Airport's minutes latitude
  7. latitude seconds - (Integer) Airport's seconds latitude
  8. latitude direction - (Text) N or S
  9. longitude degree - (Integer) Airport's degrees longitude
  10. longitude minutes - (Integer) Airport's minutes longitude
  11. longitude seconds - (Integer) Airport's seconds longitude
  12. longitude direction - (Text) E or W
  13. altitude - (Integer) Airport's altitude in feet

Reading Airport Data Files:

  • The data files have the .dat extension, which means they were created using the File data type
  • The File data type is a class that includes member functions for opening and closing a file, and also for reading the following types: Boolean, Character, Float, Integer, LongInteger, and Text
  • .dat data files must be opened and read using the File's member functions
  • The filenames are of the form AirportDataX.dat - the X indicates how many airport data records are contained in the file, various sizes are provided for testing purposes
  • The first item in the file is an integer which indicates how many records are contained in the file - this integer must be read using member function read from the File data type
  • Each field in a record must be read one at a time from the file by using one call to a read member function - so reading 1 record from an AirportDataX.dat file requires calling read 13 different times
  • For example code that illustrates how to read records using the File data type see: Building C++ Open Records
